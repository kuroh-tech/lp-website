# DF CONNECT LP修正指示書 for Opus 4.6

## 0. この修正の目的

現行LPを、Web制作会社・広告代理店が見ても違和感のないB2B向けLPに調整してください。
「情報商材っぽい」「大きな会社っぽく見せすぎている」「実績を盛って見せている」印象を消し、**個人事業主が直接対応する、問い合わせしやすい外部制作パートナー**として見せることが目的です。

LPなので、基本的に下層ページは作りません。
**LP内から遷移してよい別ページは、品質確認用のデモサイト3件のみ**です。

---

## 1. 絶対条件

### 1-1. 実績の扱い

MSCreate関連の実績は、LP上では**基本的に匿名実績として扱う**。

LP上に出してはいけないもの：

- M's Create / MSCreate の社名
- 掲載元URL
- エンドクライアント名
- エンドクライアントURL
- 実案件のロゴ
- 実案件のスクリーンショット
- 「直請け」「自社制作実績」と誤解される表現

LP上に出してよいもの：

- 業種カテゴリ
- 担当範囲
- 実施内容
- CMS対応の有無
- レスポンシブ対応の有無
- 公開時期のざっくり表記
- 協力案件であること

### LP上の表記方針

NG：

```text
株式会社メディア・ワン様
沖縄料理 龍の介様
M's Create様の制作案件に協力
掲載元で実績を見る
```

OK：

```text
協力実績：企業サイトのCMS移設・レスポンシブ対応
協力実績：店舗サイトのHTML再構築・CMS対応
制作会社様の案件にて、実装・構築の一部を担当しました。
守秘義務の都合上、社名・URLは掲載していません。
```

## 2. LP全体の見せ方

### 目指す印象

- 営業色が強すぎない
- 制作会社の現場感がある
- 相談前の心理的ハードルが低い
- 個人事業主が直接対応することが自然に伝わる
- 実績は匿名だが、担当内容は具体的に見える
- デモサイトで品質確認できる
- 「綺麗なLP」より「問い合わせしやすいLP」

### 避けたい印象

- 情報商材LP
- 成果保証っぽい訴求
- 大量受注できる制作会社のような見せ方
- 実績を盛って見せている印象
- 「完全」「圧倒的」「丸投げ」「売上アップ」などの強い売り文句
- フッターやヘッダーに下層ページがたくさんあるような見せ方

この順番を基本にしてください。

1. ヘッダー
2. ファーストビュー + 問い合わせフォーム
3. 制作会社の現場で詰まりやすい作業
4. DF CONNECTについて
5. 対応内容
6. 匿名の協力実績
7. デモサイトで品質確認
8. 進め方

11. 最終CTA

## 4. ヘッダー修正

### 4-1. ヘッダーの目的

ヘッダーは「会社サイトっぽいナビゲーション」ではなく、LP内の確認ポイントへすぐ移動するためのパーツにしてください。

下層ページへのリンクは置かないでください。
デモサイト以外の下層ページへ誘導しないでください。

### 4-2. ヘッダー左側

ロゴは以下のように見せる。

```text
DF CONNECT
個人事業の制作パートナー
```

または、横幅が足りない場合：

```text
DF CONNECT
制作会社向け外部パートナー
```

「株式会社っぽい」「大規模チームっぽい」見せ方にしない。
ロゴ周りに、小さく個人事業・外部制作パートナーであることを添える。

### 4-3. PCヘッダーナビ

ページ内アンカーのみ。

```text
対応内容
匿名実績
デモ
進め方
FAQ
```

CTA：

```text
概要だけ送る
```

リンク先：

```text
#contact
```

### 4-4. モバイルヘッダー

モバイルは詰め込みすぎない。

推奨：

```text
DF CONNECT
[概要だけ送る] [menu]
```

メニュー展開時：

```text
対応内容
匿名実績
デモ
進め方
FAQ
相談する
```

### 4-5. ヘッダーから削るもの

以下はヘッダーに置かない。

- 会社概要
- サービス詳細
- 特定商取引法に基づく表記
- プライバシーポリシー
- ブログ
- 実績一覧ページ
- 無料相談
- 資料請求

プライバシーポリシーはフォーム直下またはフッター内のLP内アンカーで扱う。
別ページにしない。

## 6. 問い合わせフォーム修正

### 6-1. フォーム見出し

```text
案件概要だけで相談できます
```

### 6-2. フォーム横コピー

```text
正式な仕様書がなくても大丈夫です。
「Figmaあり」「WordPressの固定ページ追加」「公開前チェックだけ」など、分かる範囲でお送りください。

クライアント名やURLは、NDA前であれば伏せたままで問題ありません。
```

### 6-3. フォーム項目

必須：

- 会社名・屋号
- お名前
- メールアドレス
- ご相談内容

任意：

- 案件概要・メッセージ

### 6-4. ご相談内容の選択肢

```text
コーディング・LP実装
WordPress制作・改修
運用保守・軽微修正
公開前チェック・フォーム確認
NDA・貴社名義での進行相談
まだ決まっていない
```

「ホワイトラベル相談」よりも「貴社名義での進行相談」の方が自然。

### 6-5. フォーム直下の補足

```text
送信前にクライアント名やURLを出す必要はありません。
内容を確認し、対応可否・確認したい点・概算の進め方を返信します。
```

### 6-6. ハニーポット項目

現状、クローラー上で `Website` という項目が見えている可能性があります。
スパム対策のハニーポットなら、画面上では確実に非表示にしてください。

実装方針：

- `display:none` だけでなく、アクセシビリティ上も不要な読み上げが起きないようにする
- 一般ユーザーには見せない
- フォームの自然な流れを邪魔しない

## 7. DF CONNECTについて

### セクション見出し

```text
個人事業の外部制作パートナーとして、代表者本人が窓口になります。
```

### 本文

```text
DF CONNECTは、Web制作会社様・広告代理店様の外部制作パートナーとして活動している個人事業です。
大人数の制作会社ではありません。

その分、相談内容の把握、対応可否の判断、制作時の確認、納品前のチェックまで、連絡経路をシンプルに進められます。

大量案件を同時に受ける体制ではないため、対応範囲・納期・連絡ルールを事前に整理したうえで、無理のない形で対応します。
```

### カード文言

#### 代表者本人が相談窓口

```text
営業担当と制作者が分かれていないため、相談内容のズレを減らしやすい体制です。
```

#### 対応可否を先に整理

```text
短納期や仕様が固まりきっていない案件でも、対応できる範囲と確認が必要な点を先に整理します。
```

#### 貴社の進行ルールに合わせる

```text
メール、チャット、タスク管理、ファイル共有など、発注元企業様の既存フローに合わせて進めます。
```

#### 守秘前提で相談可能

```text
NDA締結前は、クライアント名やURLを伏せた概要のみで問題ありません。
```

## 12. 対応外・要相談

### 見出し

```text
対応できる範囲を、事前に整理します。
```

### 本文

```text
個人事業として対応しているため、無理にすべてを引き受けるのではなく、対応できる範囲と難しい範囲を事前に確認します。
```

### 原則対応外

```text
大規模なシステム開発
会員機能・決済機能・予約システムなどの新規開発
仕様が未整理のままの丸投げ案件
短納期かつ仕様変更が多い案件
契約前の機密情報共有
発注元企業様の許可がない実績公開
エンドクライアントへの直接営業
```

### 要相談

```text
既存サイトの状態が不明な改修
急ぎの公開作業
複数ページの短納期対応
MTG同席が必要な案件
保守範囲が広い月額対応
```

## 13. FAQ

### Q1. 個人事業主でも、制作会社の外部パートナーとして継続依頼できますか？

```text
はい、可能です。
ただし、大人数で大量案件を同時に対応する体制ではないため、対応範囲・納期・稼働状況を事前に確認します。
継続案件の場合も、無理のない範囲を整理したうえで進めます。
```

### Q2. NDA締結前に相談できますか？

```text
はい。
NDA前は、クライアント名・URL・ログイン情報などを伏せた概要だけで問題ありません。
詳細共有が必要な場合は、NDA締結後に進めます。
```

### Q3. エンドクライアントとのやり取りもお願いできますか？

```text
原則として、エンドクライアントとの契約・連絡窓口は発注元企業様でお願いします。
必要に応じて、貴社の制作パートナーとしてMTGに同席し、技術説明や仕様確認をサポートすることは可能です。
```

### Q4. 貴社名義での納品に対応できますか？

```text
対応可能です。
表記ルール、連絡経路、共有範囲、納品形式を事前に確認し、発注元企業様の方針に合わせて進めます。
```

### Q5. デモサイトは実際の納品実績ですか？

```text
いいえ。
デモサイトは、制作品質や導線設計を確認していただくためのサンプルサイトです。
実案件の詳細は、守秘義務に配慮し、公開可能な範囲で個別にご説明します。
```

### Q6. 実績は見られますか？

```text
公開できる範囲の協力実績を、匿名で掲載しています。
社名・URL・画面キャプチャは、守秘義務や発注元企業様の方針により掲載していません。
品質確認は、デモサイトをご確認ください。
```

### Q7. どのくらいの情報があれば相談できますか？

```text
最初は、作業内容、希望納期、支給素材の有無だけで大丈夫です。
たとえば「Figmaあり、LPコーディング希望、6月上旬公開予定」程度の内容でも対応可否を確認できます。
```

## 14. 最終CTA

### 見出し


## 15. フッター修正

### 15-1. フッターの目的

フッターは、会社サイトのような大量リンクを置く場所ではなく、**最低限の事業者情報と問い合わせ導線を置く場所**にしてください。

個人事業主のLPとして、必要情報をシンプルに見せる。
「会社概要ページがありそう」「特商法ページがありそう」「下層ページが多そう」という印象を消す。

### 15-2. フッターに残すもの

```text
DF CONNECT
個人事業のWeb制作パートナー

Web制作会社・広告代理店向けに、コーディング、WordPress改修、軽微修正、公開前チェックを必要な範囲で対応しています。
```

事業者情報：

```text
屋号：DF CONNECT
事業形態：個人事業
代表者：[代表者名]
所在地：[掲載できる範囲で記載]
連絡先：お問い合わせフォームよりご連絡ください
対応領域：Web制作・改修・運用保守
```

フッター内リンク：

```text
対応内容
匿名実績
デモ
進め方
相談する
プライバシーポリシー
```

ただし、すべてLP内アンカーにする。
プライバシーポリシーもLP内の折りたたみ、モーダル、またはページ下部セクションにする。

コピーライト：

```text
© DF CONNECT
```

### 15-3. フッターから削るもの

以下は削除してください。

```text
会社概要
特定商取引法に基づく表記
大きなフッターロゴ画像
長すぎるフッター説明文
サービス一覧風の大量リンク
外部リンク集
ブログリンク
実績一覧リンク
```

### 15-4. 特定商取引法リンクについて

このLPでは、固定価格の商品をオンラインで即時購入させる構成ではなく、問い合わせ・個別見積り導線が中心です。
そのため、フッターに「特定商取引法に基づく表記」を大きく出す必要はありません。

もし今後、オンライン決済・月額商品の即時申込・固定価格商品の販売を行う場合は、別途表示を検討してください。
今回のLPでは削除で進めてください。

### 15-5. プライバシーポリシー

問い合わせフォームがあるため、プライバシーポリシー自体は必要です。
ただし、下層ページではなくLP内で完結させてください。

実装案：

- フォーム直下に「プライバシーポリシー」を折りたたみで表示
- フッターの「プライバシーポリシー」は `#privacy` へアンカー
- 別ページリンクにはしない

## 17. デザイン・UI方針

### 全体

- 派手なグラデーションを避ける
- 過剰なアニメーションを避ける
- 料金カードを目立たせすぎない
- CTAを煽らない
- 余白は確保しつつ、同業者が情報を拾いやすい密度にする
- ビジュアルよりテキストの読みやすさを優先する

### カードUI

- 実績カードは匿名前提で、担当内容が読める構成にする
- デモカードは実績カードとデザインを少し分ける
- 実績 = 守秘前提の協力案件
- デモ = 品質確認用のサンプル
- この違いが一目で分かるようにする

### CTA

- CTA文言は「問い合わせ」より「概要だけ送る」に寄せる
- CTA周辺に「社名・URLを伏せてOK」を置く
- ヘッダーCTA、FV CTA、最終CTAはすべて `#contact` に飛ばす
